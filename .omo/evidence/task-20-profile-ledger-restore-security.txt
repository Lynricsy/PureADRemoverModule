## Fix summary
- `crates/puread-cli/src/lock.rs`
  - Added explicit final lock-file symlink preflight before opening `run/puread.lock` with `O_NOFOLLOW`.
  - Existing parent chain safe-create still rejects symlinked `run/`.
  - Existing `lock_path` containment still rejects override lock paths outside module root.
- `crates/puread-cli/src/profile_execute/profile_runtime.rs`
  - Added append preflight for `state/profile-actions.jsonl`: module-root containment, safe parent creation, symlinked parent rejection, final ledger symlink rejection.
  - `JsonlProfileLedger::append` now uses the same preflight before appending.
- `crates/puread-cli/src/profile_execute/profile_restore/ledger.rs`
  - Added safe read preflight for persisted profile restore ledger: module-root containment, existing parent component checks, final ledger symlink rejection, then `O_NOFOLLOW` open.
  - Replaced JSON indexing when marking records restored with object parse + `insert`.
- `crates/puread-cli/src/profile.rs` and `profile_execute.rs`
  - `apply-profile --execute` now preflights the profile ledger once after lock acquisition and before any Android profile runner call when the plan includes appops/component/ROM actions.
- `crates/puread-cli/src/profile_execute/profile_restore.rs` and `report.rs`
  - `profile-restore --execute` still writes JSON/text report, but returns nonzero if any restore action failed.
- `crates/puread-cli/src/cli.rs`
  - Debug-only apply-profile test seam args grouped into `ApplyProfileTestArgs` to keep the CLI arg structure cleaner and satisfy clippy without suppressing warnings.

## Direct CLI env fake-runner negative check
Command used a temp module ledger and ran:
`PUREAD_TEST_PROFILE_RUNNER=1 target/debug/puread-cli profile-restore --execute --module-root <module> --profile-runner-log <runner.log>`

Result:
- JSON report showed `failed: 1` with real `/system/bin/cmd appops ...` unavailable error.
- Stderr: `profile restore failed for 1 action(s)`.
- `exit=1`.
- `runner_log_exists=no`.

Conclusion: ordinary CLI execution cannot fake success through `PUREAD_TEST_PROFILE_RUNNER`; the fake runner remains gated by hidden `--test-profile-runner`.
